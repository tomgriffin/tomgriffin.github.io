Should I have used css classes instead of id's?

I needed to create a blog. It felt like a really big job and I didn't know where to start. I decided to copy as much as I could take inspiration from a magazine that was beside me on the couch at home. At first I thought - sweet, a couple of colours on each page and the rest of the design looks quite simple.

Earlier that day, I'd gone through the Udacity course Intro to HTML and CSS. In 20. Code, Test, Refine, they suggest a process for building web pages. I really liked that big-to-small flow and I decided I'd follow it to build the main page of the blog. 1. Look for natural boxes 2. Look for repeated styles & semantic elements 3. Write your HTML 4. Apply styles (from BIGGEST to smallest) 5. Fix things Unfortunately, they didn't say how to add styles ... ID's or classes. So I sketched out a basic design using paper and pencil and started following the big-to-small flow. Once I finally got to the fun part - styling my pages like the magazine, I was really, really eager to get stuck in. I had my html page and added my title, navigation links, and some text about me. I quickly wanted to get the CSS going so I could start styling. To style the blog title _T.Here_, the quickest thing I could think of was to give the

and

each an id so I could put the ID in the CSS file and start associating styles to it. I didn't have the patience to think about what classes I'd need - because I didn't know how the actual design would look. ID's uniquely identify an element on a page. You can’t/shouldn’t have two elements with the same ID. Classes classify one or more elements in the same way. So … you kind of need to know what those classifications are first.

It took about an hour of finding colours, fonts, changing font size, aligning to the centre and fiddling with the padding before I was happy enough. So should I have used Classes rather than ID's? Let’s compare the two and think about the trade offs in using one or the other.
