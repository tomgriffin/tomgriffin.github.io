Should I have used css classes instead of id's?

I needed to create a blog. It felt like a really big job and I didn't know where to start. I decided to copy as much as I could take inspiration from a magazine that was beside me on the couch at home. At first I thought - sweet, a couple of colours on each page and the rest of the design looks quite simple.

Earlier that day, I'd gone through the Udacity course Intro to HTML and css. In 20. Code, Test, Refine, they suggest a process for building web pages. I really liked that big-to-small flow and I decided I'd follow it to build the main page of the blog.

  1. Look for natural boxes
  2. Look for repeated styles & semantic elements
  3. Write your HTML
  4. Apply styles (from BIGGEST to smallest)
  5. Fix things
Unfortunately, they didn't say how to add styles ... especially whether to use id's or classes.

So I sketched out a basic design using paper and pencil and started following the big-to-small flow.

Once I finally got to the fun part - styling my pages like the magazine, I was really, really eager to get stuck in. I had my html page and added my title, navigation links, and some text about me. I quickly wanted to get the css going so I could start styling.

To style the blog's title T.Here, the quickest thing I could think of was to give the <div> and <h1> elements each an id so I could put the id in the css file and start associating styles to it. I didn't have the patience to think about what classes I'd need - because I didn't know how the actual design would look.
  • Id's uniquely identify an element on a page. You can’t/shouldn’t have two elements with the same id.
  • Classes classify one or more elements in the same way. So … you kind of need to know what those classifications are first.

It took about an hour of finding colours, fonts, changing font size, aligning to the centre and fiddling with the padding before I was happy enough.

So should I have used Classes rather than id's? Let’s compare the two and think about the trade offs in using one or the other.

Comparison point ID's Classes Creating my first blog page Maintaining & collaborating on the blog in the future
Syntax html: <h1 id="hero-title">
css: #hero-title {} 		html: <h1 class="hero-title">
css: .hero-title {}		 Syntax differences are trivial Syntax differences are trivial
Scope Identify a single element and need to be unique Classify a group of elements I was working on styling one element at a time. So ID’s were great because any changes would only affect the element with that ID.

But if I wanted to style 2 or 3 elements in the same way e.g. items in the top nav, I’d need to repeat the style for each element 		It’ll be too hard to keep ID’s unique going forward.

I’ll definitely want to style elements in common groups and classes are the best way to do this. Apply the style to the class and all elements in the class will get updated.

In the end, I think ID’s were fine for an initial hack-together of a design on a single page. For any other purpose, classes will be better.
